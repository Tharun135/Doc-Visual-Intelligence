Doc Visual Intelligence - User Guide

👋 Welcome!

This tool analyzes your technical documents and suggests where to add visual aids like diagrams, screenshots, and flowcharts to improve readability.

🚀 Quick Start (3 Steps)

  1. Upload or paste your document - Supports PDF, DOCX, TXT, MD, JSON (max 25MB)
  2. Click "Analyze Visual Opportunities" - The tool scans your entire document using configurable detection rules
  3. Review recommendations - See what visuals to add and where

📄 How to Upload Your Document

Option 1: File Upload

  • Click "File Upload" tab on the left
  • Drag & drop your file OR click to browse
  • Supported formats: PDF, DOCX, TXT, MD, JSON
  • Maximum file size: 25MB

Option 2: Paste Text

  • Click "Pasted Text" tab
  • Copy your text and paste into the editor
  • Perfect for testing small sections

Option 3: Sample Templates

  • Try pre-loaded samples to see how recommendations work
  • Click any sample button: "Sync Procedure", "System Architecture", etc.

📊 Understanding the Results

Left Panel: Sections List

  • Section title - The heading or section number from your document
  • Visual badge - Shows recommended visual type (e.g., "Flowchart", "Diagram")
  • Text snippet - Preview of the section content
  • Click any section - Details appear on the right

Right Panel: Detailed Recommendations

  • Visual Type - What to create (Screenshot, Flowchart, etc.)
  • Reader Benefit - Why this visual helps readers
  • Reader Value (1-5) - How much effort it saves readers
  • Confidence % - How strongly the detection rules matched this section
  • Priority - High, Medium, Low importance
  • Recommended Placement - Where in the text to add it

🎨 Types of Visuals Recommended

📸

Screenshot

Show UI elements, menus, or configuration panels to help users navigate

🌳

Visualize logical choices and outcomes

📝

🎬

GIF Tutorial

Create screen recordings for multi-step procedures

✅ Working with Recommendations

Feedback

  • Rate recommendations: "Was this useful?" with 👍 Yes or 👎 No
  • Your feedback helps improve rule accuracy for future analyses

Advanced Details

  • Click "Advanced Details" to see signal analysis
  • View which rules triggered this recommendation and why
  • See confidence scores and detected content type

Capture Specifications

  • Screenshot recommendations include a "Capture Specification" checklist
  • Use Purpose, Include, Primary Focus, and Exclude fields to align screenshots across pages
  • Copy insertion snippets to add placeholders to your document
  • The tool can generate Mermaid diagram code automatically from detected content

💡 Tips & Tricks

  • Long documents? - Click sections on the left to jump to recommendations instantly (no scrolling needed!)
  • Test first - Try sample templates to understand how recommendations work
  • Check placement hints - "After step 3:" tells you exactly where to add the visual
  • Reader value matters - Focus on high value recommendations (4-5 dots) first
  • High priority sections - These typically improve readability most
  • Multiple recommendations? - One section can have several visual suggestions

❓ FAQ

What formats can I upload?

PDF, DOCX (Word), TXT, MD (Markdown), and JSON files up to 25MB

How does it detect sections?

The tool identifies headings (numbered like "1.2.3" or markdown "#"), and structural keywords like "Procedure", "Overview", "Introduction"

Can I edit recommendations?

Recommendations are rule-based suggestions. You control what actually gets added to your document

Why "No recommendation"?

Some sections may be best as pure text (for example, summaries or disclaimers). The tool respects this and returns no recommendation

What does confidence % mean?

Higher % = more detection rules matched this section. 80%+ is a strong match

Can I regenerate the analysis?

Yes! Upload the same or different document anytime to get fresh recommendations

📞 Need More Help?
